`ModelState` — это специальный словарь, который хранит как сами привязанные значения, так и ошибки, возникшие на этапе привязки типов или при проверке атрибутов валидации (DataAnnotations)

**ModelState.IsValid**
  Свойство, которое возвращает `true`, если все данные корректно привязались к типам и прошли валидацию
  Возвращает `false`, если есть хотя бы одна ошибка
  В API-контроллерах (с атрибутом `[ApiController]`) эта проверка происходит под капотом автоматически

**Nullable типы и необязательные параметры**
  Использование знака вопроса `?` делает параметр *необязательным* для привязчика
  Если клиент не пришлет данные для `int?` или `DateTime?`, переменная просто получит значение `null`, и ошибка в `ModelState` добавлена не будет

**Атрибут [Required] и значимые типы (Value Types)**
  Значимые типы (например, `int` или `double`) не могут быть `null`
  Если клиент не пришлет такое поле в JSON, система просто подставит дефолтное значение (`0`), и проверка `[Required]` будет успешно пройдена, так как ноль не является пустотой
  Чтобы заставить привязчик строго требовать наличия числа от клиента, свойство специально делают Nullable: `[Required] public int? Age { get; set; }`
  В таком случае отсутствие данных приведет к значению `null`, и атрибут `[Required]` честно выдаст ошибку валидации

**Извлечение списка ошибок (Model.Errors)**
  Если автоматическая генерация ответа 400 Bad Request отключена или не подходит, коллекцию ошибок можно извлечь вручную из свойства `ModelState.Values`

```csharp
[HttpPost]
public IActionResult CreateUser([FromBody] UserDto user)
{
    if (!ModelState.IsValid)
    {
        // Извлекаем все сообщения об ошибках в плоский список
        var errors = ModelState.Values
            .SelectMany(v => v.Errors)
            .Select(e => e.ErrorMessage)
            .ToList();

        return BadRequest(new { ValidationErrors = errors });
    }

    return Ok();
}
```
