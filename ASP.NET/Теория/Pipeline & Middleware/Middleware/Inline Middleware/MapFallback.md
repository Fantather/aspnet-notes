Группа методов расширения для `IEndpointRouteBuilder`, предназначенная для обработки запросов, которые не были сопоставлены ни с одним другим определенным маршрутом (Endpoint).

Является частью механизма Endpoint Routing и имеет самый низкий приоритет выполнения.

### Назначение
* **Поддержка Single Page Applications (SPA)**
  В SPA-фреймворках (React, Angular, Vue) маршрутизация часто происходит в браузере (Client-side routing). Например, URL `/settings` не соответствует физическому файлу на сервере. `MapFallback` позволяет вернуть входной файл приложения (обычно `index.html`) для любых неизвестных путей, чтобы JavaScript мог загрузиться и обработать маршрут самостоятельно.

* **Централизованная обработка ненайденных ресурсов**
  Используется для перенаправления на специальный контроллер или страницу вместо возврата ошибки 404, если запрос не соответствует ни одному API-эндпоинту или статическому файлу.


**Варианты методов**
* **MapFallbackToFile**
  Возвращает указанный статический файл (обычно из папки `wwwroot`). Наиболее часто используется для SPA.

* **MapFallbackToPage**
  Перенаправляет запрос на указанную Razor Page.

* **MapFallbackToController**
  Перенаправляет запрос на действие MVC контроллера.

* **MapFallback**
  Принимает делегат `RequestDelegate` для произвольной обработки.

**Технические детали**
- _Приоритет:_ Endpoint'ы, созданные через `MapFallback`, всегда проверяются в последнюю очередь.

- _Статус код:_ В отличие от обработки 404 ошибок, `MapFallback` обычно возвращает статус 200 OK и содержимое файла/страницы, так как запрос считается успешно обработанным (просто другим ресурсом)


### Пример использования (Сценарий SPA)
```csharp
var builder = WebApplication.CreateBuilder(args);
var app = builder.Build();

// 1. Попытка отдать статический файл (js, css, png)
app.UseStaticFiles();

app.UseRouting();

// 2. Обработка API запросов (высокий приоритет)
app.MapControllers(); 

// 3. Fallback (низкий приоритет)
// Если запрос не совпал с файлом и не совпал с API -> отдаем index.html
app.MapFallbackToFile("index.html");

app.Run();
````
