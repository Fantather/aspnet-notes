**Razor** — это механизм шаблонизации (View Engine), используемый в ASP.NET Core для создания динамических веб-страниц. Он позволяет смешивать стандартную HTML-разметку с кодом на языке C#.


## Основные концепции
**Серверный рендеринг (SSR)**
  Razor выполняется *на сервере*.
  Когда поступает запрос, сервер выполняет C#-код внутри Razor-файла, генерирует итоговую HTML-строку и отправляет её браузеру.
  Браузер получает только чистый HTML (без C# кода).

**Расширение .cshtml**
  Файлы Razor имеют расширение `.cshtml` (C# + HTML).

**Компиляция**
  При сборке или запуске приложения файлы `.cshtml` компилируются в обычные C# классы.
  Это обеспечивает высокую производительность и проверку типов (IntelliSense).


## Где используется Razor
**ASP.NET Core MVC**
  В качестве "Views" (Представлений) для контроллеров.
  Лежат в папке `Views`.

**Razor Pages**
  В качестве страниц в странично-ориентированной архитектуре.
  Лежат в папке `Pages`.

**Blazor**
  В качестве компонентов для интерактивных веб-приложений (здесь Razor используется немного иначе, для создания дерева рендеринга, а не просто HTML-строки).

## Ключевая особенность (@)
Вся магия Razor строится вокруг символа `@`.
Этот символ говорит движку: "Здесь заканчивается HTML и начинается C#".

*Пример:*
```html
<p>Текущее время:</p>

<p>@DateTime.Now</p>
