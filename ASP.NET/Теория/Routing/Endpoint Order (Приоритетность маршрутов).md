## Описание механизма
В системе маршрутизации ASP.NET Core (Endpoint Routing) выбор маршрута происходит на основе алгоритма «наилучшего совпадения» (Best Match). По умолчанию система автоматически вычисляет приоритет (Precedence) на основе специфичности шаблона URL.

Свойство `Order` позволяет вручную вмешаться в этот процесс и задать абсолютный приоритет выполнения, игнорируя стандартные правила специфичности шаблонов.


Использование `Order` для разрешения конфликтов маршрутизации считается крайней мерой. 
В хорошо спроектированном API маршруты должны различаться по уникальным шаблонам URL или HTTP-методам, а не полагаться на скрытые числовые приоритеты, которые сложно отлаживать.


## Принцип работы Order
**Числовое значение**
  Свойство принимает целое число (`int`). *Правило инвертировано: чем меньше числовое значение, тем выше приоритет маршрута.* Эндпоинт с `Order = -1` будет рассматриваться раньше, чем эндпоинт с `Order = 0`.

**Значение по умолчанию**
  Если свойство не задано явно, оно равно `0`.

**Алгоритм выбора**
  При входящем запросе маршрутизатор группирует все зарегистрированные эндпоинты:
  1. Сначала выбираются эндпоинты с самым низким значением `Order`.
  2. Внутри группы с одинаковым `Order` применяется стандартное сравнение шаблонов (Template Precedence). *Например, литерал `/api/user` выиграет у параметра `/api/{id}` только если у них одинаковый Order.*

## Синтаксис и реализация
```csharp
// Регистрация эндпоинта с модификацией Order
app.MapGet("/some-route", handler)
   .Add(endpointBuilder => 
   {
       // 1. Приведение типов
       // Базовый класс EndpointBuilder не содержит свойства Order.
       // Необходимо приведение к RouteEndpointBuilder.
       var builder = (RouteEndpointBuilder)endpointBuilder;

       // 2. Установка приоритета
       // Устанавливаем 1. Это значит, что данный маршрут будет иметь
       // МЕНЬШИЙ приоритет, чем стандартные маршруты (у которых 0).
       builder.Order = 1;
   });
````

### Где это применяется
**Catch-all маршруты (Жадные маршруты)**
При создании маршрутов вида `/{**slug}` (например, для CMS), которые перехватывают всё. _Им необходимо явно задавать `Order` выше (например, 1 или 99), чтобы они проверялись только после того, как система убедилась, что запрос не соответствует конкретным API-эндпоинтам или статическим файлам (у которых Order = 0)._
