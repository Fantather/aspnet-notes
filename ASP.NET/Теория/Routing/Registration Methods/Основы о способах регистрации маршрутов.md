Регистрация маршрутов в ASP.NET Core выполняется через методы расширения интерфейса `IEndpointRouteBuilder` в файле `Program.cs`. Эти методы создают связь между входящим URL (включая HTTP-метод) и обработчиком запроса (эндпоинтом)

Существует три основных подхода к регистрации, каждый из которых предназначен для своих архитектурных сценариев


### Основные подходы
**HTTP методы (Minimal API)**
  Используют методы `MapGet`, `MapPost`, `MapPut`, `MapDelete`.
  Логика обработки пишется внутри делегата (лямбды) прямо в месте регистрации маршрута.
  _Применяется для:_ Микросервисов, легковесных API, приложений без сложной структуры классов.

**Атрибутная маршрутизация (Attribute Routing)**
  Использует метод `MapControllers`.
  Маршруты задаются декларативно с помощью атрибутов `[Route]`, `[HttpGet]` непосредственно над контроллерами и методами действий.
  _Применяется для:_ REST API, где важен точный контроль над URI ресурсов.

**Конвенциональная маршрутизация (Conventional Routing)**
  Использует метод `MapControllerRoute`.
  Маршруты определяются глобальными шаблонами (шаблонами путей), которые сопоставляют сегменты URL с именами контроллеров и методов.
  _Применяется для:_ MVC-приложений (Monolith) с HTML-представлениями (Views).


### Пример подключения в Program.cs
```csharp
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddControllersWithViews(); // Сервисы для MVC и API

var app = builder.Build();

// 1. Minimal API
app.MapGet("/status", () => "Running");

// 2. Атрибутная маршрутизация (для REST API контроллеров)
app.MapControllers();

// 3. Конвенциональная маршрутизация (для MVC контроллеров)
app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");

app.Run();
