Записывает логи в файл

#### rollingInterval
`rollingInterval` - интервал времени, через который логи начнут сохраняться в новый файл
- `RollingInterval.Day` - раз в день, используется чаще всего, стоит по умолчанию
- `RollingInterval.Mounth` - если логи сохраняются редко
- `RollingInterval.Hour` - если логи сохраняются часто
- `RollingInterval.Infinite` - логи не будут самостоятельно создавать новый файл

#### retainedFileCountLimit
`retainedFileCountLimit` - количество лог-файлов, которые могут хранится одновременно
Автоматически удаляет более старые лог файлы
Это защищает диск от переполнения

#### fileSizeLimitBytes и rollOnFileSizeLimit
`fileSizeLimitBytes` и `rollOnFileSizeLimit: true` - максимальный размер лог-файла и настройка, при достижении максимального размера, создавать новый лог-файл
Вместе это защищает нас от лог-файла размером в несколько гигабайт

#### outputTemplate
Указываем, как именно будет выглядеть строка Логов
Мы можем вставлять в логи любые символы `[]`, `()`, `-` и т.д., что бы сделать его читаемым

`{Timestamp:yyyy-MM-dd HH:mm:ss.fff} [{Level:u3}] {Message:lj}{NewLine}{Exception}`
- **`{Timestamp: ...}`**:
    - `Timestamp` — это свойство "Время события".
    - `:yyyy-MM-dd HH:mm:ss.fff` — это **форматирование** (стандартный синтаксис .NET). Вы можете его сократить, например, до `:HH:mm:ss`, если не хотите видеть дату.

- **`[{Level:u3}]`**:
    - `Level` — это свойство "Уровень" (Info, Error и т.д.).
    - `:u3` — это форматирование. `u` означает UpperCase (ВЕРХНИЙ РЕГИСТР), а `3` — 3 символа. Это превращает `Information` в `INF`, `Warning` в `WRN`, `Error` в `ERR`, `Fatal` в `FTL`. Очень удобно для выравнивания.

- **`{Message:lj}`**:
    - `Message` — это ваше текстовое сообщение.
    - `:lj` — это форматирование. `l` означает "literal" (просто текст), `j` — говорит ему красиво отформатировать JSON, если он вдруг окажется в сообщении. Обычно можно просто оставить `{Message}`.

- **`{NewLine}`**:
    - Просто вставляет перенос строки (`\r\n`).

- **`{Exception}`**:
    - Если вы передали в лог исключение (например, `Log.Error(ex, ...)`), здесь будет напечатан его **полный стек вызовов (`.ToString()`)**. Если исключения не было, это свойство будет пустым


##### Примеры кода
```C#
.WriteTo.File(
    "logs/log-.txt", // Имя файла, будет записывать логи в папку
                     // тире в конце заставит его создавать 
                     // файлы "log-20251108.txt"
	
	outputTemplate: "{Timestamp:yyyy-MM-dd HH:mm:ss.fff} [{Level:u3}] {Message:lj}{NewLine}{Exception}", // Форматирование строки логов
	
    rollingInterval: RollingInterval.Day, // 1. Ротация (Rolling)
    
    retainedFileCountLimit: 7, // 2. Очистка
    
    fileSizeLimitBytes: 10 * 1024 * 1024, // 3. Лимит размера (10 МБ)
    
    rollOnFileSizeLimit: true // 4. Создать новый, если лимит превышен
)
```
