Это вызов делегата, это метод который выполняет тот метод, на который ссылается делегат
Этот метод генерируется компилятором для каждого типа делегата
Статически типизированный и самый быстрый способ вызвать делегат
Компилятор проверяет типы параметров

Например для `public delegate int Transformer(int x);` существует метод `int Invoke(int x)` на экземпляре делегата

Когда вы пишете `myDelegate(arg1, arg2)`, компилятор под капотом превращает это в `myDelegate.Invoke(arg1, arg2)`. То есть прямой вызов делегата эквивалентен вызову `Invoke`, НО
- вызов `Invoke` позволяет понять что мы используем делегаты, делая код более читаемым
- Оператор `?.` проверяет `MyEvent` на `null` и только затем вызывает `Invoke`, рекомендуется использовать с `Events`

```C#
Action a = () = Console.Writeline("hi");
a();          // компилятор -> a.Invoke();
a.Invoke();   // явно
```

```C#
Func<int, int> f = x => x * 2;
int res = f(5);         // -> f.Invoke(5)
int res2 = f.Invoke(5);
```

```C#
MyEvent?.Invoke(this, EventArgs.Empty);
```
