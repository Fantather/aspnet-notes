Стандартный тип делегата для событий
Пустой класс, от которого должны наследоваться классы, которые будут хранить информацию передаваемую вместе с событием, потому что
- Этого общепринятая практика и это ожидается
- `EventHandler<TEventArgs>`, то есть с параметрами, в качестве дополнительной информации для события ожидает класс который наследуется от `EventArgs`
  Обычный `EventHandler` не имеет особых требований

**Ожидается что**
- Класс не будет содержать методов
- Название класса будет заканчиваться на `EventArgs`
- Поля класса не будут иметь `set`, потому что событие уже произошло и изменение передаваемой информации уже не нужно и может привести к багам

Если дополнительная информация не требуется, вместо класса передают `EventArgs.Empty`
```C#
public class InformationEventArgs(int value) : EventArgs
{
	public int Value { get; } = value;
}
```
