Это событие, которое прерывает нормальное выполнение программы

Исключение сигнализирует о какой-то ошибке или необычной ситуации, которую программа не может обработать сама

На пример:
- Деление на ноль
- Попытка выйти за пределы массива
- Ошибка при открытии файла


**Команды**
*try*
Оборачивает код, который может вызвать исключение (ошибку во время выполнения)

*catch*
Ловит (обрабатывает) исключение, которое произошло в блоке `try`
`catch (Extption ex)` - отловит все ошибки

*finally*
Код, который выполняется всегда, независимо от того, произошло исключение или нет
Обычно используется для освобождения ресурсов (Файловые потоки, соединение с БД)

*throw*
Создаёт и "выбрасывает" исключение


**Вложенные исключения**
Внутри `try` можно написать ещё один `try` `catch`
Внутренний `catch` может обработать исключение сам, если исключение не было отловлено, его попытается перехватить внешний `catch`

**Повторное генерирование исключения**
Если `catch` отловил исключение, но мы хотим чтобы его обработали выше, мы можем передать текущее исключение выше

Это делается с помощью вызова `throw;`, но без указания объекта, внутри блока `catch`
Это не создаёт новое исключение, а передаёт текущее


***Примеры***
Обычное использование
```C#
// Оборачиваем код который может вызвать исключение
try
{
    int x = 10 / 0;
    int y = 0;
    
    if (y == 3)
	    throw new ArgumentException("y не может быть равен 0");
}

catch (DivideByZeroException ex)
{
    Console.WriteLine("Деление на ноль невозможно!");
}

finally
{
    file.Close(); // закроем файл в любом случае
}
```

Вложенные исключения
```C#
try
{
    try
    {
        int x = 10 / 0;
    }
    catch (FormatException) { }
}
catch (DivideByZeroException)
{
    Console.WriteLine("Деление на ноль поймано во внешнем блоке");
}

```

Повторное генерирование исключения
```C#
catch (Exception)
{
    Console.WriteLine("Ошибка поймана, но пробрасываем выше");
    throw; // повторно бросаем исключение
}
```
`throw;`  используется внутри `catch` для повторного выброса текущего исключения
Оно не создаёт новое исключение, а передаёт текущее
