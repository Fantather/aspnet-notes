`Extension-методы` - это способ расширить функциональность уже существующего типа, не изменяя его код

**Условия**
- Метод должен быть статическим и быть в статическом **классе**
- Первый параметр выглядит как `this T`, где `T` это тип который мы расширяем

**Ограничения**
- У класса расширения не должно быть внутреннего состояния (Полей, Свойств)
- Нельзя расширять функциональность Типа с помощью статических структур и их методов, компилятор ожидает класс
- Работает только для типов в нашей Области видимости

В коде мы вызываем этот код из расширенного типа, но под капотом Компилятор вызывает наш статический объект и из него наш метод, так что это синтаксический сахар

```C#
// Объявление
public static class StringExtensions
{
	public static bool IsCapitalized(this string str)
	{
		if (string.IsNullOrEmpty(str)) return false;
		else return char.IsUpper(str[0]);
	}
}

// Использование
string word = "Word";
bool result = string.IsCapitalized(word);

// Под капотом это выглядит как
bool result = StringExtensions.IsCapitalized(word);
```
