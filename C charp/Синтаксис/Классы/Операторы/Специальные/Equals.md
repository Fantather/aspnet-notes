Проверяет два объекта на равенство, возвращает `bool`

Переопределяется во встроенных типах данных
По умолчанию проверяет, указывают ли переменные на один и тот же объект в памяти

В ссылочных типах, проверяет тождественность ссылок, а не их содержимого

```C#
bool s = obj1.Equals(obj2);
```


**Перегрузка**
При переопределении `Equals` нужно обязательно переопределить `GetHasCode`, `==` и `=!`, что бы хеш-алгоритмы работали корректно

```C#
public override bool Equals(object? obj)
{
    if(obj is not Employee other) return false;
    return this.Salary == other.Salary;
}
```

- `obj is not Employee other` — проверяет, **не** является ли `obj` объектом класса `Employee`
- Если `obj` **не** является `Employee`, то условие истинно — и метод выполняет `return false`
- Если же `obj` **является** `Employee`, то оператор `is not` не срабатывает, и переменная `other` получает этот объект в типизированной форме внутри текущей области
  То есть `other` становится переменной типа `Employee`
