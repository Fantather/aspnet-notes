## Виды записей (C# 10+)
**record class**
  То же самое, что и просто `record`. Это ссылочный тип, живущий в куче (Heap). Используется по умолчанию.

**record struct**
  Это *значимый тип* (value type), живущий в стеке. Сочетает легковесность структур и удобство записей (сравнение, `ToString`, деконструкция).

```csharp
// Свойства мутабельны (изменяемы) по умолчанию для struct
public record struct Point(int X, int Y);

// Чтобы сделать неизменяемым:
public readonly record struct ImmutablePoint(int X, int Y);
````


## Когда использовать
**DTO (Data Transfer Objects)**
Идеально подходят для объектов, которые просто переносят данные между слоями приложения или через API.

**Конфигурации**
Для хранения настроек, которые загружаются один раз и не должны меняться.

**Ключи словарей**
Благодаря корректной реализации `GetHashCode` и `Equals` "из коробки", записи отлично подходят в качестве сложных ключей для `Dictionary` или `HashSet`.

**Функциональный стиль**
Если вы пишете код без побочных эффектов, используя иммутабельные данные.


## Отличия от Class и Struct

| **Характеристика**   | **Class**      | **Record (Class)**        | **Struct**                | **Record Struct** |
| -------------------- | -------------- | ------------------------- | ------------------------- | ----------------- |
| **Тип памяти**       | Heap (Куча)    | Heap (Куча)               | Stack (Стек)              | Stack (Стек)      |
| **Сравнение (`==`)** | По ссылке      | _По значению_             | Ошибка (надо перегружать) | _По значению_     |
| **Изменяемость**     | Обычно Mutable | Обычно Immutable (`init`) | Mutable                   | Mutable           |
| **Наследование**     | Поддерживает   | Только от Record          | Нет                       | Нет               |
