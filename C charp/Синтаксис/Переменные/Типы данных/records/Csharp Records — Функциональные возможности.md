## Основные возможности
**Неизменяемость (Immutability)**
  По умолчанию позиционные записи создают свойства с модификатором `init`. Это значит, что значение можно задать только при создании объекта. Это делает код потокобезопасным и предсказуемым.


**Сравнение по значению (Value Equality)**
  Компилятор автоматически переопределяет методы `Equals()`, `GetHashCode()` и операторы `==` / `!=`.

```csharp
var person1 = new Person("Alice", "Smith", 25);
var person2 = new Person("Alice", "Smith", 25);

// Вернет true, так как данные совпадают
Console.WriteLine(person1 == person2); 

// Для обычного класса (class) это вернуло бы false
````


**Неразрушающее изменение (Nondestructive Mutation)**
Так как записи часто неизменяемы, для "изменения" используется ключевое слово `with`. Оно создает _новый_ объект, копируя данные из старого и изменяя указанные свойства.

```C#
var person1 = new Person("Alice", "Smith", 25);

// Создает копию person1, но с другим именем
var person2 = person1 with { FirstName = "Bob" }; 
```


**Встроенное форматирование**
Компилятор переопределяет метод `ToString()`. Он выводит не просто имя типа, а список всех свойств и их значений.

```C#
// Вывод: Person { FirstName = Alice, LastName = Smith, Age = 25 }
Console.WriteLine(person1);
```


**Деконструкция (Deconstruction)**
Позиционные записи автоматически поддерживают деконструкцию в кортежи.

```C#
var person = new Person("Alice", "Smith", 25);

// Разбираем объект на переменные
var (first, last, age) = person;
```
