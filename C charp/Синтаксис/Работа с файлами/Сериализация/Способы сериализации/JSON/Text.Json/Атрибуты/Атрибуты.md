**Управление именем**
Изменяет имя свойства в файле, на переданное
```C#
[JsonPropertyName("customerName")]
public string FullName { get; set; }
```


**Полное игнорирование**
Помечает свойство, которое должно быть полностью проигнорировано и при сериализации (не попадёт в JSON) и при десериализаци (даже если оно есть в JSON, будет проигнорировано)
```C#
[JsonIgnore]
```


**Управление политиками NULL**
Чаще настраивается глобально в опциях, для всего сериализатора, но использовать атрибуты так же возможно
```C#
// Всегда игнорировать (аналог [JsonIgnore])
[JsonIgnoreCondition(JsonIgnoreCondition.Always)]
public string? InternalId { get; set; }

// Игнорировать только если значение null
[JsonIgnoreCondition(JsonIgnoreCondition.WhenWritingNull)]
public string? Description { get; set; }

// Никогда не игнорировать (поведение по умолчанию)
[JsonIgnoreCondition(JsonIgnoreCondition.Never)]
public string? Name { get; set; }
```

**Включение / Исключение Полей / Свойств**
`[JsonInclude]` / `[JsonExclude]` - для включения/исключения непубличных полей и свойств

```C#
[JsonInclude] // Это private поле будет сериализовано!
private string _internalId;

[JsonExclude] // Это public поле НЕ будет сериализовано!
public string TemporaryCode;
```


**Обработка числовых значений**
Актуально для работы с большими числами в JSON, которые могут приходить в виде строк (Частая практика в JavaScript для больших чисел)

```C#
[JsonNumberHandling(JsonNumberHandling.AllowReadingFromString | 
                    JsonNumberHandling.WriteAsString)]
public class Product
{
    public string Name { get; set; }
    public decimal Price { get; set; } 
    // Price можно прочитать из строки, и будет записан как строка
    public long BigNumber { get; set; }
}

string json = @"{""Name"":""Book"",""Price"":""29.99"",""BigNumber"":""123456789012345""}";
var product = JsonSerializer.Deserialize<Product>(json);

json = JsonSerializer.Serialize(product);
// {"Name":"Book","Price":"29.99","BigNumber":"123456789012345"}
```
**Доступные флаги:**
- `Strict` - Поведение по умолчанию
- `AllowReadingFromString` - Разрешить десериализацию числа из строки JSON
- `WriteAsString` - Сериализовать числовое свойство как строку JSON
- `AllowNamedFloatingPointLiterals` - Разрешить `NaN`, `Infinity`, `-Infinity` в JSON
