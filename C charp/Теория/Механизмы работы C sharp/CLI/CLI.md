*Common Language Infrastucture* - Открытый стандарт (ECMA-335, ISO/IEC 23271), описывающий среду выполнения для кода на различных языках программирования

**Ключевые компоненты CLI**
- [[CTS|CTS]] (Common Type System)
  Определяет общее представление типов данных для всех языков .NET (например, C#, F#, VB.NET), обеспечивая их совместимость

- [[CLS|CLS]] (Common Language Specification)
  Набор правил, которым должен следовать компилятор языка, чтобы создавать компоненты, совместимые с другими языками .NET

- [[VES]] (Virtual Execution System)
  Это виртуальная исполняющая система, которая обеспечивает среду выполнения для программ, скомпилированных в [[IL-код|CIL]]

- [[IL-код|CIL]]/[[IL-код|IL]] (Common Intermediate Language)
  Промежуточный язык, в который компилируется исходный код любого языка .NET
  Этот код затем компилируется Just-In-Time (JIT) в машинный код во время выполнения

- [[C charp/Теория/Механизмы работы C sharp/CLI/CLR/Метаданные|Метаданные]]
  Подробная информация о структуре кода (классы, методы, свойства), хранящаяся в сборках вместе с CIL



CLI (**Common Language Infrastructure**) - это **стандарт** (описание на бумаге), утверждённый ECMA/ISO.  
В нём прописано:

- какие бывают типы (CTS)

- какие правила для межъязыковой совместимости (CLS)

- как устроены метаданные, сборки, IL-код

- и как виртуальная машина (VES) должна всё это выполнять
