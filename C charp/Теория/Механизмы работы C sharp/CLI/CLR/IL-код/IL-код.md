*Common Intermediate Language* (IL, CIL, MSIL) - Это не зависящий от процессора машинный язык, который используют .NET программы

Компиляторы .NET языков компилируют исходный код в IL, который потом JIT-компилятор компилирует в машинный код

***Преимущества***
IL - язык более высокого уровня по сравнению с большинством других *машинных* языков

- **Не зависит от архитектуры процессора или ОС**
  Может выполняться на любой машине, если есть CLR для неё
  
- **Безопасный**
  CLR следит за *Безопасностью типов*, чтобы операции выполнялись только с теми данными, с которыми они совместимы
  
 - операции могут выполняться только над совместимыми типами данных
   нельзя сложить число и ссылку на объект
   
 - методы и свойства можно вызывать только у объектов соответствующего типа
   нельзя вызвать метод для массива у строки
   
 - преобразования типов допускаются только если они допустимы
  
  
   CLR следит за *Безопасностью использования памяти*, это означает что 
   - доступ к памяти возможен только в пределах допустимых границ (Нельзя выйти за пределы массива)

   - каждый объект изолирован и его данные не могут быть изменены извне (Нельзя случайно залезть в память другого объекта)

   - ссылки на объекты всегда указывают на корректные области памяти, исключая работу с «висячими» или недействительными указателями

**Особенности**
- Стековый язык
  Большинство инструкций работает через стек (push/pop)
- Объектно-ориентированный машинный язык
  Поддерживает классы, интерфейсы, наследование, виртуальные методы
- Поддерживает вызов и отлов исключений

**Возможности IL**
Имейте в виду, что любой язык высокого уровня, скорее всего, использует лишь часть возможностей, предоставляемых CLR. При этом язык ассемблера IL открывает доступ ко всем возможностям CLR. Если выбранный вами язык программирования не дает доступа именно к тем функциям CLR, которые необходимы, можно написать часть программного кода на ассемблере IL или на другом языке программирования, позволяющем их задействовать

- Работа с **типами** (примитивы, объекты, массивы, generics)
- Создание/инициализация объектов (`newobj`)
- Вызов методов (обычных, статических, виртуальных)
- Управление потоком: циклы, переходы (`br`, `switch`, `leave`, `throw`)
- Исключения: `try`, `catch`, `finally`, `throw`
- Операции с памятью и стеком (`ldloc`, `stloc`, `ldarg`, `starg`).
- Работа с массивами (`newarr`, `ldelem`, `stelem`)
- Метаданные: IL тесно связан с таблицами метаданных .NET (информация о типах, методах, сборках)
