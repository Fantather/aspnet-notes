*Common Language Specification* - это подмножество правил [[CTS|CTS]], набор стандартов, которым должны следовать разработчики компиляторов и библиотек, чтобы обеспечить межъязыковую совместимость между различными языками .NET

То что допустимо в CTS, может быть недопустимо в CLS

CLS гарантирует что код написанный на одном .NET-совместимом языке, может беспрепятственно использован из любого другого .NET языка без проблем совместимости или непредвиденных ошибок

То есть IL-код полученный из одного .NET языка сможет быть корректно использован любым другим компилятором .NET языка, потому что не будет опираться на особенности только одного языка

Эти правила и ограничения касаются только Публичной части API

**Примеры правил**
- Идентификаторы не должны отличаться только регистром
  Потому что некоторые языки (например, Visual Basic .NET) не различают регистр символов
  Объявить два метода `Calculate()` и `calculate()` в одном классе — нельзя

- Запрет на использование беззнаковых типов (uint, ulong) в публичном API
  Не все языки .NET (например, ранние версии J#) поддерживают беззнаковые типы
  Поэтому вместо `uint` в публичных методах следует использовать `int`

- Типы параметров в перегруженных методах должны существенно отличаться
  Перегрузка методов должна быть однозначной для всех языков

- Нельзя использовать не безопасный код
