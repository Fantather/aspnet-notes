*Абстрактный класс*  это класс, объявленный с модификатором `abstract`

Его главная особенность в том, что *нельзя создавать экземпляры абстрактного класса напрямую* (через `new`)
Он существует только для того, чтобы быть базовым для других классов

**Абстрактные классы используются для**:
- *Создания шаблона (контракта)*: Абстрактный класс задаёт общую структуру и частичную реализацию для группы тесно связанных классов
  Он определяет, _что_ должны делать производные классы (через абстрактные члены), и часто предоставляет, _как_ это делать частично (через реализованные методы)

- *Принудительного полиморфизма*: Абстрактные методы _обязывают_ производные классы предоставить их реализацию. Это гарантирует, что у всех потомков будет определённый набор методов

- *Общего кода*: Абстрактный класс позволяет вынести общую логику из всех производных классов в одно место, избегая дублирования кода


Абстрактный класс может включать *обычные (неабстрактные) методы или поля* с готовым кодом. 
В этом случае класс даёт не только "контракт", но и готовую общую логику, которую наследники могут использовать
```C#
abstract class Shape
{
    public string Name { get; set; }

    // Обычный метод с кодом — готовая часть реализации:
    public void PrintInfo()
    {
        Console.WriteLine($"{Name}: Area={Area()}, Perimeter={Perimeter}");
    }

    // Абстрактные члены — «что должны уметь»:
    public abstract double Area();
    public abstract double Perimeter { get; }
}
```
