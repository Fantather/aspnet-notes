*Не обобщённые типы* - Это обычные классы, структуры, методы и интерфейсы, где тип данных *жёстко задан*

Все новые библиотеки, фреймворки и сам .NET ориентированы на [[Generic|generics]]
*Non-generic* коллекции (`ArrayList`, `Hashtable`) оставлены ради совместимости с очень старым кодом


**Недостатки**
Могут работать с несколькими типами сразу через упаковку/распаковку, но у этого подхода есть минусы
- *Падение производительности* (из-за упаковки/распаковки)
- *Нет строгой типизации*: можно случайно положить строку в коллекцию чисел - и ошибка проявится только во время выполнения


**Может использоваться**
Когда нужно хранить элементы *разных типов* в одной коллекции (например, и числа, и строки вперемешку).  
Но даже тогда обычно делают `List<object>`, что уже *generic*

Этот `Box` работает *только с int*
Если тебе нужен `Box` для строки - придётся писать новый класс
```C#
public class Box {
    public int Value { get; set; }
}
```
