Спецификаторы доступа в C# — это ключевые слова, которые определяют, где и кем может использоваться класс, метод, поле или другой член кода. Они контролируют **уровень видимости и доступности** элементов программы, обеспечивая инкапсуляцию и безопасность.

---

### Основные спецификаторы доступа в C#

|Спецификатор|Где доступен|Применяется к|
|---|---|---|
|`public`|Везде — из любого кода|Классы, методы, поля и др.|
|`private`|Только внутри текущего класса|Члены класса, вложенные типы|
|`protected`|В текущем классе и в его наследниках|Члены класса|
|`internal`|В пределах текущей сборки (проекта)|Классы, члены|
|`protected internal`|В пределах текущей сборки или наследниках|Члены класса|
|`private protected`|В пределах текущей сборки и наследниках|Члены класса|

---

### Что дают спецификаторы доступа?

- **Скрывают детали реализации** от других частей программы (инкапсуляция).
    
- **Защищают данные** от неправильного использования.
    
- **Позволяют чётко структурировать код** по уровням доступа.
    
- **Помогают избежать ошибок и конфликтов** при масштабировании проекта.
    

---

### Краткие примеры

```csharp
public class Person
{
    private string ssn;           // Видно только в классе Person
    protected string name;        // Видно в Person и наследниках
    internal int age;             // Видно внутри сборки
    public void Speak() { }       // Доступно всем
}
```
