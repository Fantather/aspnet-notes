### Игнорирование свойств (NotMapped / Ignore)


- **[[Attribute|Атрибут]]  `[NotMapped]`**
    
    - Используется для свойств, которые **не должны создаваться в таблице БД**.
        
    - EF Core будет **игнорировать** это свойство при построении схемы и миграций.
-
```c#
public class Company
{
    public int Id { get; set; }
    public string Name { get; set; }

    [NotMapped]
    public string TempData { get; set; } // не сохраняется в БД
}
```

**[[Entity FM/Урок 2 04.09.25 (заполнить название дз)/Способы настройки моделей/FluentApi/FluentApi]] `Ignore`**

- То же самое, но через Fluent API, если не хочется ставить атрибут в модели:
``` c#
modelBuilder.Entity<Company>()
    .Ignore(c => c.TempData);
```
- **Когда использовать:**
    
    - Временные вычисляемые свойства
        
    - Свойства для UI или логики, не нуждающиеся в хранении
        
    - Любые данные, которые **не должны влиять на БД**
        
- **Итог:**  
    `[NotMapped]` и `Ignore` просто **убирают свойства из метаданных БД**, но свойства остаются в модели C# для работы в коде.
