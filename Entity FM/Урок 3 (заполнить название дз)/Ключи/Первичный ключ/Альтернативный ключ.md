**Определение:**  
Альтернативный ключ — это **уникальный ключ, который не является первичным**, но гарантирует уникальность значений в колонке или наборе колонок.
![[Pasted image 20250905195658.png]]
#### Одинарный альтернативный ключ

- **Когда использовать:**  
    Когда есть поле, которое должно быть уникальным, но не является PK.
    
- **Пример через Data Annotations:**
```c#
public class Employee
{
    public int Id { get; set; } // PK

    [Required]
    [Index(IsUnique = true)] // только EF6, для EF Core — через Fluent API
    public string Email { get; set; } // альтернативный ключ
}

```

Пример через Fluent API (EF Core):
```c#
modelBuilder.Entity<Employee>()
    .HasAlternateKey(e => e.Email);
```
#### Складовой (Composite) альтернативный ключ

- **Когда использовать:**  
    Когда уникальность определяется **несколькими колонками одновременно**.
    
- **Пример Fluent API:**
```c#
modelBuilder.Entity<Employee>()
    .HasAlternateKey(e => new { e.FirstName, e.LastName });
```

- В БД создаётся **уникальный индекс** на комбинацию `FirstName + LastName`.
