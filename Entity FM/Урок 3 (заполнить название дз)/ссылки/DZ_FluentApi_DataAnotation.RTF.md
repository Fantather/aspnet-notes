# Опис

Необхідно створити базу даних **«Академія» (Academy)**, яка міститиме інформацію про співробітників та внутрішній порядок академії.

- **Викладачі**, що читають лекції в академії, представлені у вигляді таблиці **«Викладачі» (Teachers)**, в якій зібрана основна інформація: ім'я, прізвище, дані про зарплату та дата прийому на роботу.
    
- **Групи** зберігаються в таблиці **«Групи» (Groups)**.
    
- **Факультети** та **кафедри** представлені у таблицях **«Факультети» (Faculties)** та **«Кафедри» (Departments)** відповідно.
    

**Використані технології та підходи:**

- CodeFirst
    
- Налаштування моделей: **DataAnotation [Attributes]** та **Fluent API**
    
- Змінення іменування полів таблиць по принципу `назватаблиці_назва_поля`.  
    Наприклад: поле `Id` у таблиці `Departments` буде мати назву в БД — `departments_id`.
    
- **Migration**
    
- Заповнення таблиць даними через `DbContext`
    
- CRUD (INSERT, UPDATE, DELETE, SELECT) через відповідні `DbSet`

# Таблиці

Нижче наведено детальний опис структури кожної таблиці.

---

## 1. Групи (Groups) → `study_groups`

- **Ідентифікатор (Id_Groups)**
    
    - Унікальний ідентифікатор групи
        
    - Тип даних: `int`
        
    - Авто збільшення
        
    - Не може містити `null`
        
    - Первинний ключ
        
- **Назва (Name)**
    
    - Назва групи (накладаємо складений альтернативний ключ: Id + Name)
        
    - Тип даних: `varchar(10)`
        
    - Не може містити `null`
        
    - Не може бути порожнім
        
    - Має бути унікальним
        
- **Рейтинг (Rating)**
    
    - Рейтинг групи
        
    - Тип даних: `int`
        
    - Не може містити `null`
        
    - Має бути в діапазоні від 0 до 5
        
- **Курс (Year)**
    
    - Курс (рік навчання) групи
        
    - Тип даних: `int`
        
    - Не може містити `null`
        
    - Має бути в діапазоні від 1 до 5

## 2. Кафедри (Departments)

- **Ідентифікатор (Id)**
    
    - Унікальний ідентифікатор кафедри
        
    - Тип даних: `int`
        
    - Авто збільшення
        
    - Не може містити `null`
        
    - Первинний ключ
        
- **Фінансування (Financing)**
    
    - Фонд фінансування кафедри
        
    - Тип даних: `money`
        
    - Не може містити `null`
        
    - Не може бути менше ніж 0
        
    - Значення за замовчуванням: 0
        
- **Назва (Name)**
    
    - Назва кафедри
        
    - Тип даних: `nvarchar(100)`
        
    - Не може містити `null`
        
    - Не може бути порожнім
        
    - Має бути унікальним

## 4. Викладачі (Teachers)

- **Ідентифікатор (Id)**
    
    - Унікальний ідентифікатор викладача
        
    - Тип даних: `int`
        
    - Авто збільшення
        
    - Не може містити `null`
        
    - Первинний ключ
        
- **Дата працевлаштування (EmploymentDate)**
    
    - Дата прийому викладача на роботу
        
    - Тип даних: `date`
        
    - Не може містити `null`
        
    - Не може бути меншим ніж 01.01.1990
        
- **Ім'я (Name)**
    
    - Ім'я викладача (накласти альтернативний ключ)
        
    - Тип даних: `nvarchar(max)`
        
    - Не може містити `null`
        
    - Не може бути порожнім
        
- **Надбавка (Premium)**
    
    - Надбавка викладача
        
    - Тип даних: `money`
        
    - Не може містити `null`
        
    - Не може бути менше ніж 0
        
    - Значення за замовчуванням: 0
        
- **Ставка (Salary)**
    
    - Ставка викладача
        
    - Тип даних: `money`
        
    - Не може містити `null`
        
    - Не може бути менше або рівним 0
        
- **Прізвище (Surname)**
    
    - Прізвище викладача
        
    - Тип даних: `nvarchar(max)`
        
    - Не може містити `null`
        
    - Не може бути порожнім
