# Сравнение React, Angular и Vue.js

###  React

- **Тип:** Библиотека (для построения пользовательских интерфейсов).
    
- **Маршрутизация (Routing):** **Отсутствует встроенная.** React отвечает только за "View" (отображение).
    
- **Экосистема:**
    
    - Для маршрутизации нужно выбирать стороннюю библиотеку (стандарт де-факто: `react-router-dom`).
        
    - Для управления состоянием тоже нужны сторонние решения (Redux, Zustand, MobX).
        
- **Философия:** "Learn once, write anywhere". Полная свобода выбора инструментов, но требует самостоятельной настройки архитектуры проекта.
    

###  Angular

- **Тип:** Полноценный фреймворк (платформа).
    
- **Маршрутизация (Routing):** **Встроена.** Поставляется "из коробки" (`@angular/router`).
    
- **Экосистема:**
    
    - Всё включено: работа с формами, HTTP-клиент, валидация, маршрутизация — это части самого Angular.
        
    - Использует TypeScript принудительно.
        
    - Использует RxJS (реактивное программирование) для работы с данными.
        
- **Философия:** "All-in-one". Жесткая структура проекта, много правил, но не нужно искать библиотеки для базовых задач. Высокий порог входа.
    

###  Vue.js (то, что вы назвали VoidJs)

- **Тип:** Прогрессивный фреймворк.
    
- **Маршрутизация (Routing):** **Вынесена в отдельный пакет, но является официальной.**
    
- **Экосистема:**
    
    - В отличие от React, команда Vue официально поддерживает и развивает ключевые библиотеки: `vue-router` (маршрутизация) и `pinia` (ранее Vuex, управление состоянием).
        
    - Вам не нужно выбирать из десятка роутеров, есть один официальный.
        
- **Философия:** Взял лучшее от React и Angular.
    
    - От **React**: Виртуальный DOM, компонентный подход, однонаправленный поток данных.
        
    - От **Angular**: Удобные шаблоны (разделение HTML и JS), директивы (v-if, v-for), двустороннее связывание данных.
        
- **Почему "меньше чем Angular, больше чем React":** Ядро Vue маленькое (как у React), но наличие официальных инструментов делает его более структурированным, не превращая в такого гиганта, как Angular.
