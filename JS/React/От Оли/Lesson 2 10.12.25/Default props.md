`defaultProps` – это возможность в React задать значения по умолчанию (fallback values) для свойств компонента (`props`). Эти значения будут использованы, если родительский компонент не передаст соответствующее свойство, либо если передаст значение `undefined`.

пишется через [[Деструктуризация||деструкторизацию]]
### пособы использования `defaultProps`

С развитием React и появлением функциональных компонентов (Functional Components) с хуками, подход к заданию значений по умолчанию изменился.

![[Pasted image 20251210185531.png]]

В современных функциональных компонентах предпочтительный способ — использование **деструктуризации с параметрами по умолчанию** (ES6 Default Parameters)

```jsx
// Предпочтительный и современный способ
const Button = ({ type = 'button', text = 'Нажми меня' }) => {
  return <button type={type}>{text}</button>;
};
```
- **Пояснение:** В этом случае, если свойство `type` или `text` не будет передано при использовании компонента, будут автоматически использованы значения, указанные в деструктуризации.


___
#### Для классовых компонентов (Class Components)

В классовых компонентах `defaultProps` задается как статическое свойство класса.
![[Pasted image 20251210185649.png]]
нужно использовать .defaultProps

(новый синтаксис)
```jsx
class Button extends React.Component 
{ static defaultProps = 
{ type: 'button', text: 'Нажми меня' }; // ... 
}
```

___
### Сравнение подходов

|**Метод**|**Компонент**|**Преимущества**|**Недостатки (Особенности)**|
|---|---|---|---|
|**Деструктуризация (ES6)**|Функциональный|Современный стандарт, лаконичность, хорошая интеграция с TypeScript.|**Не работает** для старых классовых компонентов.|
|**`Component.defaultProps`**|Классовый/Функциональный|Унифицированный подход, работает с `propTypes`.|**Устаревает** для функциональных компонентов в новых версиях React.|
В React 18 и более поздних версиях для функциональных компонентов **рекомендуется** использовать деструктуризацию с параметрами по умолчанию, как наиболее чистый и идиоматический подход.
