### Классический подход (Inline HTML)

Старый способ, когда обработчик вешается прямо в HTML-атрибут.

![[Pasted image 20251210191743.png]]

**Минусы:**

1. **Ограниченность:** Невозможно (или очень трудно) добавить несколько разных обработчиков на одну кнопку.
    
2. **Сложность тестирования:** Логика тесно связана с разметкой.
    
3. **Сложность поддержки:** При масштабировании трудно вносить изменения в разбросанный по HTML код.
___
### Использование `addEventListener` и делегирования

Чтобы обойти минусы инлайн-стиля, разработчики стали использовать `addEventListener`. Для оптимизации часто используется паттерн **"Делегирование событий"** (вешается один обработчик на родителя, который ловит всплывающие события от детей).

```JavaScript
  document.getElementById("parent").addEventListener("click", function (event) {

    const button = event.target;

    switch (button.id) {

      case "ok":

        console.log("OK!");

        break;

      case "cancel":

        console.log("Cancel");

        break;

    }
```
**Проблемы этого подхода:**

- **Кроссбраузерность:** В разных браузерах (особенно старых) реализация событий может отличаться.
    
- **Много шаблонного кода (Boilerplate):** Приходится писать много лишних проверок (`event.target`, `switch/case`), чтобы определить, какой именно элемент был нажат.
___
### Подход React

React объединяет удобство написания с производительностью, используя **Synthetic Events** (Синтетические события).

- **Делегирование:** React под капотом вешает один глобальный обработчик событий на корневой элемент приложения (Event Delegation), что экономит память.
    
- **Синтаксис:** Вы пишете обработчики прямо в JSX, используя **camelCase** (например, `onClick`, `onChange`), что выглядит просто, как в "классическом подходе", но работает эффективно.
    
- **Совместимость:** React использует обертку над нативными событиями, гарантируя одинаковое поведение во всех браузерах.
- 
![[Pasted image 20251210191949.png]]

реакт использует метод делегирования и используется обычный обработчик через camel case
___

```jsx
import React from "react";

function App() {

  const handleClick = (event) => {

    const id = event.target.id;

    switch (id) {

      case 'ok':

        console.log('OK!');

        break;

      case 'cancel':

        console.log('Cancel');

        break;

      default:

        console.log('Нажата неизвестная кнопка');

    }

  };

  return (

    <div onClick={handleClick}>

      <h2>Делегирование событий в React</h2>

      <p>React упрощает работу с событиями и избавляет от кучи шаблонного и несовместимого кода.</p>

      <button id="ok">OK</button>

      <button id="cancel">Cancel</button>

    </div>

  );

}

export default App;
```

# Искуственые события

реакт исползает их что бы ликвидировать несовместимость браузеров

api для событий единый для всех браузеров и работает даже для старых версий

для всех событий в реакте используется кемел кейс
```jsx
import React from "react";

function App() {

  const handleEvent = (event) => {

    event.preventDefault();       // предотвращает стандартное поведение (если нужно)

    event.stopPropagation();      // предотвращает всплытие (если нужно)

    console.log(

      "Это также означает, что API для отмены событий един во всех браузерах; другими словами, методы event.stopPropagation() и event.preventDefault() работают даже в старых версиях Internet Explorer.\n" +

      "Синтаксис упрощает совместную поддержку элементов UI и обработчиков событий. Это похоже на традиционные встроенные обработчики событий, но в действительности все обстоит иначе. На самом деле в целях повышения производительности React использует делегирование событий.\n" +

      "Для обработчиков событий React задействует верблюжий регистр (camelCase), поэтому вместо onclick используется форма записи onClick."

    );

  };

  return (

    <div>

      <h1 onClick={handleEvent}>First Lesson</h1>

      <input type="text" onChange={handleEvent} placeholder="Enter text..." />

      <button onClick={handleEvent}>Click me</button>

    </div>

  );

}

export default App;
```
