#### Суть концепции

Динамическая маршрутизация позволяет использовать **один компонент** для отображения контента на множестве разных URL-адресов, имеющих общую структуру. Части URL помечаются как переменные (параметры).

#### Настройка маршрута (Route)

Для обозначения динамического сегмента используется двоеточие **`:`** перед именем параметра.

**Синтаксис:**
```JSX
<Route path="/путь/:имяПараметра" element={<Компонент />} />
```

Пример (Интернет-магазин):

Вместо создания тысяч маршрутов для каждого товара, мы создаем один шаблон.

```JSX
// App.js
<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/products" element={<ProductList />} />
  
  {/* :productId — это плейсхолдер, который примет любое значение */}
  <Route path="/products/:productId" element={<ProductDetail />} />
</Routes>
```

#### Извлечение параметров (useParams)

Чтобы компонент узнал, какой именно адрес открыт, используется хук `useParams`.

- **Импорт:** `import { useParams } from 'react-router-dom';`
    
- **Возвращаемое значение:** Объект, где ключи соответствуют именам, заданным в `<Route>` (после двоеточия).


**Пример реализации компонента:**
```JSX
// ProductDetail.js
import React, { useEffect, useState } from 'react';
import { useParams } from 'react-router-dom';

function ProductDetail() {
  // 1. Достаем параметр из URL
  // Если URL = /products/iphone-15, то productId = "iphone-15"
  const { productId } = useParams();
  
  const [product, setProduct] = useState(null);

  // 2. Используем параметр для загрузки данных (симуляция)
  useEffect(() => {
    // В реальности здесь был бы fetch(`/api/products/${productId}`)
    console.log(`Загрузка данных для товара с ID: ${productId}`);
  }, [productId]); // Важно: перезапуск эффекта при смене ID

  return (
    <div>
      <h1>Страница товара</h1>
      <p>ID товара: {productId}</p>
    </div>
  );
}
```

#### Множественные параметры
В одном URL может быть несколько динамических сегментов.

**Маршрут:**
```JSX
<Route path="/orders/:orderId/items/:itemId" element={<OrderItem />} />
```

**URL:** `/orders/555/items/abc-1`

**Результат `useParams()`:**
```JSX
{
  orderId: "555",
  itemId: "abc-1"
}
```

#### Важные нюансы ⚠️
1. **Тип данных:** Все параметры, возвращаемые `useParams`, являются **строками** (`string`). Даже если URL выглядит как `/users/42`, `userId` будет равен `"42"`. Если нужно число для математики или строгого сравнения, его нужно преобразовать: `Number(userId)`.

2. **Уникальность:** Имена параметров в вложенных маршрутах должны быть уникальными, чтобы не перезаписывать друг друга.
