Для начала убедимся, что у нас есть необходимая библиотека
В экосистеме React для веба мы используем пакет `react-router-dom`

### 1. BrowserRouter
Первым шагом мы должны подключить наше приложение к Browser History API. Для этого используется компонент `<BrowserRouter>`.

Он является **Context Provider'ом**. Это означает, что он должен находиться выше всех остальных компонентов, которым нужен доступ к маршрутизации (ссылкам, хукам навигации и т.д.). Обычно его помещают в самый корень приложения (например, в `main.tsx` или `index.js`), оборачивая компонент `<App />`.

### 2. Routes и Route
Внутри приложения мы определяем таблицу маршрутизации.

- **`<Routes>`**: Этот компонент-контейнер просматривает все свои дочерние элементы `<Route>`, находит тот, чей `path` совпадает с текущим URL, и рендерит его.

- **`<Route>`**: Определяет соответствие между путем URL (`path`) и компонентом React (`element`), который должен быть отрисован.


Взглянем на базовую структуру в файле `App.js`:
```JSX
import React from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';

// Допустим, у нас есть простые компоненты-страницы
const Home = () => <h1>Главная страница</h1>;
const About = () => <h1>О нас</h1>;

function App() {
  return (
    <BrowserRouter>
      <div className="app-container">
        <Routes>
          {/* Если URL "/", рендерим Home */}
          <Route path="/" element={<Home />} />
          
          {/* Если URL "/about", рендерим About */}
          <Route path="/about" element={<About />} />
        </Routes>
      </div>
    </BrowserRouter>
  );
}

export default App;
```

Обратите внимание на свойство `element`. В версиях React Router v6 и выше мы передаем туда именно JSX (экземпляр компонента `<Home />`), а не просто ссылку на функцию.


### Страница ошибки
Теперь давай рассмотрим важный сценарий: пользователь переходит по ссылке, которой не существует (например, `/abracadabra`). Обычно в таких случаях нужно показать страницу "404 Not Found".

Чтобы это сделать, мы добавляем ещё один `<Route>` (обычно в самый конец списка `Routes`). У него должен быть специальный `path`, который сработает, если ни один из предыдущих маршрутов не подошёл

В React Router символ `*` (звёздочка) работает как **wildcard** (универсальный шаблон). Если ни один из вышестоящих маршрутов не совпал с текущим URL, роутер выберет этот маршрут. Это идеальное место для рендеринга страницы 404

```JSX
<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/about" element={<About />} />
  <Route path="/contact" element={<Contact />} />
  
  {/* Этот маршрут сработает для любого URL, не описанного выше */}
  <Route path="*" element={<NotFound />} />
</Routes>
```
