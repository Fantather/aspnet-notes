####  Основы клиентской маршрутизации (Client-Side Routing)
В одностраничных приложениях (SPA) навигация должна происходить без повторного запроса HTML-документа с сервера. React Router реализует это через **History API** браузера.

Компонент `<Link>` Базовый примитив для декларативной навигации.

- **Функция:** Создает доступный элемент ссылки. При клике перехватывает событие, предотвращает стандартное поведение браузера (перезагрузку страницы) и обновляет URL через History API. Это инициирует перерисовку дерева компонентов React.

- **Синтаксис:**
    - Использует свойство `to` (вместо HTML-атрибута `href`) для указания целевого пути.
    
    - Рендерится в DOM как стандартный тег `<a>`.


Пример использования в коде:
```JS
import { Link } from 'react-router-dom';

function Navigation() {
  return (
    <nav>
      <Link to="/">Главная</Link>
      <Link to="/about">О нас</Link>
      <Link to="/contact">Контакты</Link>
    </nav>
  );
}
```


#### Компонент `<NavLink>`

**Что это**
`<NavLink>` — это специализированная версия компонента `<Link>`, которая обладает дополнительной логикой для сверки текущего URL браузера с путем, указанным в свойстве to.

**Зачем нужен**
Основная цель NavLink — обеспечить визуальную обратную связь (индикацию) в интерфейсе, указывая пользователю, в каком разделе приложения он находится. Это необходимо для реализации паттерна «активного пункта меню» в навигационных панелях, табах (вкладках) и хлебных крошках.

**Механизм работы**
Когда React Router определяет, что текущий URL совпадает с путем ссылки, он устанавливает внутренний флаг активности. Начиная с версии v6, доступ к этому состоянию предоставляется через Render Props (функцию, передаваемую в качестве значения пропа).

**API стилизации**
NavLink позволяет динамически вычислять классы CSS или инлайн-стили на основе состояния isActive.


*Использование с className (CSS классы)*:
Принимает функцию, которая получает объект состояния `{ isActive, isPending }`
```JSX
<NavLink
  to="/profile"
  className={({ isActive }) => isActive ? "menu-item active" : "menu-item"}
>
  Профиль
</NavLink>
```

- Если URL совпадает (`/profile`), применяется класс `"menu-item active"`.
- Если не совпадает, применяется базовый класс `"menu-item"`.


*Использование со style (Инлайн-стили)*
Аналогично принимает функцию для вычисления объекта стилей JS.
```JSX
<NavLink
  to="/settings"
  style={({ isActive }) => ({
    color: isActive ? "var(--active-color)" : "var(--default-color)",
    fontWeight: isActive ? "700" : "400"
  })}
>
  Настройки
</NavLink>
```
