#### Подход А: Папка как Компонент (`PascalCase`)

Это самый распространенный стандарт. Если папка создается для изоляции конкретного компонента (чтобы хранить там `.jsx`, `.css`, `.test.js` вместе), она называется точно так же, как и сам компонент - в `PascalCase` (Первая буква заглавная)

При импорте путь выглядит предсказуемо и совпадает с именем класса/функции компонента: `import Button from './components/Button/Button';`

```plaintext
src/
  components/
    Button/           <-- Папка в PascalCase
      Button.jsx      <-- Файл компонента
      Button.css      <-- Стили
    UserProfile/      <-- Папка в PascalCase
      UserProfile.jsx
      UserProfile.module.css
```


#### Подход Б: Группировка по доменам (`kebab-case` или `camelCase`)

Если папка служит просто контейнером для группы компонентов (как в примере учителя), часто используют строчные буквы.

- **`kebab-case`** (`admin-panel`): Очень популярно в веб-разработке, так как URL-адреса и пакеты NPM обычно пишутся в нижнем регистре через дефис.

- **`camelCase`** (`adminPanel`): Используется реже, но допустимо для соответствия стилю именования переменных в JS.

```PlainText
src/
  components/
    adminPanel/        <-- Группирующая папка (camelCase)
      StatsWidget.jsx  <-- Компонент внутри (PascalCase)
      UserTable.jsx    <-- Компонент внутри (PascalCase)
    userPanel/
      Avatar.jsx
```
