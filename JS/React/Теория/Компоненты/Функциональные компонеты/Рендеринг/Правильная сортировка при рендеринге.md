```JSX
import React, { useId, useState } from "react";
const todos = [
  { id: 1, text: "Learn React", completed: false },
  { id: 2, text: "Build a TodoList", completed: true },
  { id: 3, text: "Review JavaScript", completed: false },
];

function App() {
  return (
    <div>
       <ul>
        {todos
          .filter((todo) => todo.completed)
          .map((todo) => (
            <li key={todo.id}>{todo.text}</li>
          ))}
      </ul>
    </div>
  );
}

export default App;
```

У нас элементы сначала фильтруются, а потом мы рендерим отфильтрованные значения

Это улучшает читаемость кода и упрощает тестирование
Потому что, если через if фильтровать при рендере, то вместо неподходящих данных мы будем возвращать null
Это усложнит тестирование
