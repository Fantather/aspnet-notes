В `useReducer` у нас вся логика находится в рамках одной функции

**Преимущества**
- Помогает структурировать логику обновления состояния
- Делает код более чистым
- Предсказуемым
- Лёгким для тестирования

Идеально подходит для создания глобального состояния в сочетании с [[Context API]]


Используется вместо useState, когда состояние компонента сложное, потому что useState становится труднее поддерживать
