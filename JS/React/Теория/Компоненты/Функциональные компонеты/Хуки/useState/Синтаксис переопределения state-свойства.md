Проблема кроется в синтаксисе **стрелочных функций** (Arrow Functions) в JavaScript и том, как интерпретатор различает **тело функции** и **литерал объекта**.

В JavaScript фигурные скобки `{}` имеют двойное назначение:

1. Обозначают блок кода (тело функции, цикл, условный оператор).
    
2. Обозначают литерал объекта (создание объекта).
    

### Разбор первого варианта (Не работает)

```js
this.setState((prevState) => {
    tasks: prevState.tasks.filter((task) => task.id !== id)
});
```

Когда интерпретатор видит `{` сразу после стрелки `=>`, он считает, что открывается **тело функции** (блок кода), а не объект.

Внутри этого блока:

1. `tasks:` интерпретируется не как ключ объекта, а как **метка** (Label) — устаревшая, но валидная конструкция в JS (используется с `break`/`continue`).
    
2. Код после двоеточия выполняется как выражение.
    
3. Самое главное: в этом блоке **отсутствует ключевое слово `return`**.
    

Так как `return` нет, функция возвращает `undefined`. Когда `setState` получает `undefined`, он просто ничего не делает. Состояние не обновляется.

### Разбор второго варианта (Работает)
```js
this.setState((prevState) => ({
    tasks: prevState.tasks.filter((task) => task.id !== id)
}));
```

Здесь используются круглые скобки (), оборачивающие фигурные {}.

Круглые скобки (оператор группировки) заставляют интерпретатор воспринимать содержимое как выражение, а не как блок кода. Следовательно, { ... } распознается именно как литерал объекта.

Стрелочная функция в таком виде использует **неявный возврат** (implicit return). Она автоматически возвращает этот объект. `setState` получает объект `{ tasks: [...] }` и корректно обновляет состояние.

### Альтернативный вариант (Явный возврат)

Если вы хотите использовать первый синтаксис (без круглых скобок вокруг объекта), вам нужно явно указать `return`:

```js
this.setState((prevState) => {
    // Теперь это тело функции
    return { // Явный возврат объекта
        tasks: prevState.tasks.filter((task) => task.id !== id)
    };
});
```

### Краткий итог

- `=> { ... }` — это **блок кода**. Требует явного `return`.
    
- `=> ({ ... })` — это **возврат объекта**. Работает автоматически.

Вам стоит придерживаться второго варианта для краткости кода (`implicit return`).
