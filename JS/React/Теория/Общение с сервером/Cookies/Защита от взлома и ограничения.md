#### Ограничения (Limits)
- **Объем:** Большинство браузеров ограничивают размер одного cookie до **4096 байт** (включая имя, значение и атрибуты).
- **Количество:** Ограничение на количество cookie на один домен (обычно около 20–50 штук, зависит от реализации User Agent).

#### Векторы атак и защита
1. **XSS (Cross-Site Scripting)**
    - _Суть:_ Злоумышленник внедряет JS-код на страницу, считывает `document.cookie` и отправляет сессионные токены на свой сервер.
    - _Защита:_ Использование флага **`HttpOnly`**. Это делает cookie недоступным для `document.cookie`, оставляя его доступным только для HTTP-транспорта.

2. **CSRF (Cross-Site Request Forgery)**
    - _Суть:_ Злоумышленник заманивает авторизованного пользователя на свой сайт, который отправляет скрытый запрос к целевому сервису (например, `POST /bank/transfer`). Браузер автоматически прикрепляет cookie, и сервер выполняет операцию.
    - _Защита:_ Использование флага **`SameSite=Strict`** или **`SameSite=Lax`**.

3. **MITM (Man-in-the-Middle)**
    - _Суть:_ Перехват незашифрованного трафика для кражи cookie.
    - _Защита:_ Использование флага **`Secure`** и настройка HSTS на сервере.


#### Cookie Prefixes (Дополнительная защита)
Если имя cookie начинается с определенного префикса, браузер требует наличия конкретных атрибутов:
- `__Host-`: Должен иметь флаг `Secure`, не иметь атрибута `Domain`, `Path` должен быть `/`.
- `__Secure-`: Должен иметь флаг `Secure`.
