#### Определение
**HTTP Cookie** — это небольшой фрагмент данных (пара «ключ-значение» с набором атрибутов), который отправляется веб-сервером и хранится на стороне клиента (User Agent / Браузера).

#### Назначение
HTTP является протоколом без сохранения состояния (stateless). Cookie используются для эмуляции состояния между независимыми HTTP-запросами.

- **Управление сессиями:** Аутентификация, корзины товаров, игровые сессии.

- **Персонализация:** Пользовательские настройки (тема, язык интерфейса).

- **Трекинг:** Анализ поведения пользователя.


#### Жизненный цикл обмена данными
Процесс установки и передачи cookie происходит через HTTP-заголовки.

1. Установка (Server $\to$ Client):
Сервер отправляет ответ с заголовком Set-Cookie. Можно отправить несколько таких заголовков в одном ответе.
```HTTP
HTTP/1.0 200 OK
Content-type: text/html
Set-Cookie: session_id=xyz123; HttpOnly; Secure
Set-Cookie: theme=dark; Max-Age=3600
```

2. Хранение (Client):
Браузер сохраняет эти данные в своем хранилище cookie (Cookie Jar), ассоциируя их с доменом.

3. Отправка (Client $\to$ Server):
При каждом последующем запросе к тому же домену (с учетом правил Path и Domain), браузер автоматически извлекает подходящие пары «ключ-значение» и отправляет их в заголовке Cookie.
```HTTP
GET /dashboard HTTP/1.1
Host: www.example.com
Cookie: session_id=xyz123; theme=dark
```


> **Важно:** Атрибуты (Expires, HttpOnly и т.д.) отправляются **только** от сервера к клиенту. При возврате cookie на сервер передаются только «ключ=значение».
