## ESLint (Static Code Analysis)

Это инструмент для *статического анализа* кода. Он не запускает код, а сканирует текст программы, чтобы найти синтаксические ошибки, анти-паттерны или нарушения стилевого гайда.

**Основные функции**
  
  **Поиск ошибок до выполнения**
  Обнаруживает использование необъявленных переменных, бесконечные циклы или недостижимый код.
  
  **Обеспечение единообразия стиля**
  Принудительно заставляет использовать определенные правила (например, требовать точку с запятой или запрещать `var` в пользу `const`/`let`).
  
  **Автоматическое исправление (Auto-fix)**
  Многие нарушения (например, отступы или тип кавычек) ESLint может исправить сам при сохранении файла.

**Пример работы ESLint**
``` javascript
// Код с нарушением (ESLint выдаст ошибку: "no-unused-vars")
const calculate = (a, b) => {
  const result = a + b; // Переменная result создана, но не возвращена
  return a; 
}

// Исправленный код
const calculate = (a, b) => {
  return a + b;
}
```
