Проект запускается из консоли или терминала Visual Studio Code

**Современный вариант**
`npm create vite@latest my-app` - создание приложения под названием *my-app* с использованием последней версии *vite*
Имя должно содержать только маленькие буквы
`npm create vite@latest my-app -- --template react` - создаёт приложение, выбирая значения по умолчанию (React и JS)
Так же можно написать `--template react-ts` и выбрать *TypeScript*
`cd my-app` - переходим в созданный проект
`npm i`/`npm install` -  устанавливает пакеты в проект, создаёт папку
(На скриншоте представлен в виде "Install with npm and start now?")
`npm run dev` - запуск Веб-приложения с отладкой

**Устаревший вариант**
`npx create-react-app my-app` - создание приложения с названием *my-app*
`cd my-app` - переход в созданный проект
`npm start` - запуск веб-приложения без отладки

`npx` - это инструмент, который идёт вместе с Node.js (в составе npm)
Он позволяет запускать пакеты без установки их глобально

`npx create-react-app`
Запуск пакета *create-react-app*, если он не установлен глобально, то `npx` временно скачает его, выполнит, и удалит
Это нужно что бы
- Чтобы не засорять систему глобальными пакетами.
- Чтобы всегда запускать самую свежую версию пакета.
- Чтобы проще запускать одноразовые команды (генераторы, линтеры, билд-сценарии)
