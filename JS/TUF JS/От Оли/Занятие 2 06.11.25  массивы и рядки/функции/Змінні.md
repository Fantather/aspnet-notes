## Объявление переменных в JavaScript: var, let, const

В JavaScript эти три ключевых слова используются для объявления переменных, но они имеют ключевые различия в области видимости (scope), поднятии (hoisting) и возможности переназначения.

### `var`

`var` — это устаревший способ объявления переменных, который использовался в JavaScript с самого начала.

- **Область видимости:** **Функциональная (function scope)** или **глобальная (global scope)**.
    
    - Если `var` объявлена внутри функции, она видна только внутри этой функции.
        
    - Если `var` объявлена вне функции, она становится глобальной переменной (прикрепляется к объекту `window` в браузере).
        
    - `var` **игнорирует блочную область видимости** (циклы `for`, `if`, `while` и т.д.).
        
- **Поднятие (Hoisting):** Переменные, объявленные с помощью `var`, "поднимаются" (hoist) в начало своей области видимости (функциональной или глобальной) и **инициализируются значением `undefined`**.
    
    - Это значит, что вы можете обратиться к переменной до ее объявления, и не получите ошибку (просто `undefined`).
        
- **Переназначение и Переобъявление:**
    
    - Можно **переназначать** (менять значение).
        
    - Можно **повторно объявлять** (re-declare) в той же области видимости без ошибки.

```JavaScript
function varTest() {
  console.log(a); // Выведет: undefined (из-за hoisting)
  var a = 10;
  console.log(a); // Выведет: 10

  if (true) {
    var a = 20; // Эта та же самая переменная 'a'
    console.log(a); // Выведет: 20
  }

  console.log(a); // Выведет: 20 (значение изменилось, т.к. scope функциональный)

  var a = 30; // Повторное объявление - нет ошибки
  console.log(a); // Выведет: 30
}

varTest();
```

# let
`let` — это современный способ объявления переменных, введенный в ES6 (2015). Он пришел на смену `var` для тех случаев, когда значение переменной нужно изменять.

- **Область видимости:** **Блочная (block scope)**.
    
    - Переменная, объявленная с `let`, видна только внутри блока (`{...}`), в котором она определена (например, внутри `if`, `for`, `while` или просто пары фигурных скобок).
        
- **Поднятие (Hoisting):** Переменные `let` тоже "поднимаются", но они **не инициализируются**.
    
    - Попытка доступа к `let`-переменной до ее объявления приводит к ошибке `ReferenceError`.
        
    - Период от начала блока до объявления переменной называется **"Временная мертвая зона" (Temporal Dead Zone, TDZ)**.
        
- **Переназначение и Переобъявление:**
    
    - Можно **переназначать** (менять значение).
        
    - **Нельзя** повторно объявлять в той же области видимости.

```JavaScript
function letTest() {
  // console.log(a); // Ошибка: ReferenceError (TDZ)
  let a = 10;
  console.log(a); // Выведет: 10

  if (true) {
    let a = 20; // Это новая, другая переменная 'a' (внутри блока if)
    console.log(a); // Выведет: 20
  }

  console.log(a); // Выведет: 10 (значение не изменилось, т.к. scope блочный)

  // let a = 30; // Ошибка: SyntaxError (повторное объявление)
  
  a = 40; // Переназначение - разрешено
  console.log(a); // Выведет: 40
}

letTest();
```

### `const`

`const` (константа) — также введен в ES6. Используется для объявления переменных, значение которых **не должно изменяться** (переназначаться).

- **Область видимости:** **Блочная (block scope)** (как у `let`).
    
- **Поднятие (Hoisting):** Точно так же, как у `let` (есть "Временная мертвая зона", TDZ).
    
- **Переназначение и Переобъявление:**
    
    - **Нельзя** переназначать (менять значение).
        
    - **Нельзя** повторно объявлять.
        
    - **Требует инициализации** при объявлении (нельзя написать `const a;`).
        

**Важное замечание по `const`:** `const` делает **константной саму привязку (ссылку)**, а не значение.

- Для **примитивных типов** (числа, строки, boolean) это означает, что значение изменить нельзя.
    
- Для **объектов и массивов** (ссылочные типы) это означает, что нельзя переназначить саму переменную (например, присвоить ей новый массив), но можно **изменять содержимое** объекта или массива (добавлять/удалять элементы, менять свойства).
    

**Пример `const`:**

```JavaScript

const a = 10;
// a = 20; // Ошибка: TypeError (попытка переназначения константы)

// console.log(b); // Ошибка: ReferenceError (TDZ)
const b = 20;

// const c; // Ошибка: SyntaxError (требуется инициализация)

// Пример с объектом
const user = {
  name: "Alex"
};

// user = {}; // Ошибка: TypeError (нельзя переназначить ссылку)

// Но можно изменять содержимое объекта
user.name = "John";
user.age = 30;
console.log(user); // Выведет: { name: "John", age: 30 }
```
