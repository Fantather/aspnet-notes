**Параметры** — это переменные, которые функция "ожидает" получить при вызове. Они перечисляются в круглых скобках `()` при объявлении функции.

Этот пример показывает два разных способа работы с параметрами:

### 1. Именованные (явные) параметры

Это стандартный способ, когда вы заранее даете имена ожидаемым данным.

- **Пример:** `function display(x, y)`
    
- **Объяснение:**
    
    - Функция `display` ожидает получить два параметра, которые внутри нее будут доступны как локальные переменные `x` и `y`.
        
    - **Значения по умолчанию (старый способ):** Внутри функции есть проверка `if (y == undefined)`. Это классический (до ES6) способ задать значение по умолчанию. Если функция `display` будет вызвана только с одним аргументом (например, `display(10)`), `x` станет `10`, а `y` будет `undefined`. Проверка это "поймает" и присвоит `y` значение `x`.
        

### 2. Объект `arguments` (Неявные параметры)

Внутри **любой** (нестрелочной) функции в JavaScript доступен специальный, похожий на массив, объект `arguments`. Он содержит _все_ аргументы, которые были переданы функции при вызове, **независимо от того, сколько именованных параметров было объявлено**.

- **Пример:** `function show()`
    
- **Объяснение:**
    
    - Функция `show` объявлена без именованных параметров `()`.
        
    - **`arguments.length`**: Свойство, которое хранит _фактическое_ количество переданных аргументов (например, при вызове `show(2, 5, 10)` `arguments.length` будет `3`).
        
    - **`arguments[i]`**: Доступ к конкретному аргументу по его индексу (как в массиве). `arguments[0]` — это первый аргумент, `arguments[1]` — второй, и т.д.
        
    - **Назначение:** Этот механизм используется для создания **функций с переменным числом аргументов** (variadic functions), когда вы заранее не знаете, сколько данных в нее поступит. Функция `show` перемножает все переданные ей числа
