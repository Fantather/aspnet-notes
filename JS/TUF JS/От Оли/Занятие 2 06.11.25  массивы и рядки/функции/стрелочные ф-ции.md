## Стрелочные функции (Arrow Functions)

**Стрелочные функции (`=>`)** — это специальный синтаксис для создания функций, который отличается двумя главными вещами:

1. Он короче.
    
2. Он не имеет своего собственного контекста `this`.
    

### 1. Синтаксис

Давайте перепишем ваш пример с `display` из файла, используя стрелочную функцию.

**Ваш пример (традиционное Функциональное выражение):**



```JavaScript
let display = function(x, y) {
  let z = x * y;
  document.write(z);
};

display(6, 5); // Выведет 30
```

**Тот же код, переписанный как стрелочная функция:**



```JavaScript
let display = (x, y) => {
  let z = x * y;
  document.write(z);
};

display(6, 5); // Выведет 30
```

Как видите, мы просто заменили `function(x, y)` на `(x, y) =>`.

---

### 2. Синтаксические упрощения

Стрелочные функции позволяют сделать код еще короче в двух случаях:

**А. Неявный возврат (Implicit Return)** Если тело функции состоит **только из одного выражения** (например, `return n * n`), вы можете убрать фигурные скобки `{}` и слово `return`.

- **Традиционная функция:**
    

    
    ```JavaScript
    let square = function(n) {
      return n * n;
    };
    ```
    
- **Стрелочная функция (неявный возврат):**
    
    
	
    
    ```JavaScript
    let square = (n) => n * n; // {} и 'return' убраны
    ```
    

**Б. Один параметр** Если у функции **только один параметр**, вы можете убрать круглые скобки `()` вокруг него.

- **Было:**
    
    
    
    ```JavaScript
    let square = (n) => n * n;
    ```
    
- **Стало (без скобок):**

    ```JavaScript
    let square = n => n * n;
    ```
    
- **Без параметров:** Если у функции нет параметров, пустые скобки `()` обязательны.
    
    
    
    ```JavaScript
    let greet = () => "Hello!";
    ```
    

---

### 3. Ключевые отличия от обычных функций

Это не просто "сахар". У стрелочных функций есть фундаментальные отличия:

1. **Нет своего `this` (Лексический `this`)**
    
    - **Это самое важное отличие.** Обычные функции (`function`) получают свой собственный `this` в зависимости от того, _как_ они вызваны (как метод объекта, как обычная функция и т.д.).
        
    - Стрелочные функции **не имеют своего `this`**. Вместо этого они "берут" `this` из того окружения (scope), в котором они были созданы. Это называется **лексический `this`**.
        
    - Это решает массу проблем при работе с `this` внутри обработчиков событий или методов (например, `setTimeout` или `forEach` внутри класса).
        
2. **Нет объекта `arguments`**
    
    - У стрелочных функций **нет** встроенного объекта `arguments`.
        
    - Если вам нужно собрать все аргументы, вы должны использовать **rest-параметры**: `(...args) => { ... }`.
        
3. **Нельзя использовать как конструктор**
    
    - Стрелочную функцию **нельзя** вызвать с помощью оператора `new`. Она не предназначена для создания объектов.
