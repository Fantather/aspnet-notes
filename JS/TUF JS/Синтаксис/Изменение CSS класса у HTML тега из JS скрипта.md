Да, безусловно. Манипуляция классами через JavaScript — это стандартный способ динамического изменения стилей в DOM (Document Object Model).

Для этого в JavaScript существуют два основных свойства, доступных у любого HTML-элемента. Давайте разберем их, используя корректную терминологию.

### 1. Свойство `classList` (Рекомендуемый способ)

Это свойство возвращает объект `DOMTokenList` — коллекцию активных классов элемента. Оно предоставляет удобные методы для точечного управления классами без необходимости работать со строками вручную.

Основные методы интерфейса `classList`:

- **`.add('class-name')`**: Добавляет указанный класс к списку. Если класс уже есть, ничего не происходит (дубликаты не создаются).
    
- **`.remove('class-name')`**: Удаляет указанный класс.
    
- **`.toggle('class-name')`**: "Переключает" класс. Если класс есть — удаляет его, если нет — добавляет.
    
- **`.contains('class-name')`**: Возвращает `true` или `false` в зависимости от того, есть ли такой класс у элемента (полезно для условной логики).
    

### 2. Свойство `className` (Устаревший или специфический способ)

Это свойство позволяет получить или установить значение атрибута `class` в виде обычной строки.

- **Чтение**: Возвращает строку со всеми классами, разделенными пробелом (например, `"main-btn active"`).
    
- **Запись**: Полностью **перезаписывает** всё содержимое атрибута `class`.

Представьте, что у нас есть кнопка с уже существующим классом:

<button id="myButton" class="btn-primary">Нажми меня</button>

Если мы получим этот элемент в переменную `const btn = document.getElementById('myButton');`, какой метод свойства `classList` вы бы использовали, чтобы добавить к кнопке класс `active`, **не стирая** при этом существующий класс `btn-primary`?
