**LocalStorage** — это свойство глобального объекта `window`, предоставляющее доступ к локальному хранилищу браузера (Web Storage API). Данные хранятся в формате пар **«ключ-значение»** и сохраняются даже после перезагрузки страницы или закрытия браузера.


### 1. Основные методы интерфейса

Для управления данными используются четыре базовых метода:

- *localStorage.setItem('key', 'value')*
    Создает новую запись или обновляет существующую.
    - `key`: строка, имя ключа.
    - `value`: строка, сохраняемые данные.

- *localStorage.getItem('key')*
    Возвращает значение, связанное с указанным ключом.

- *localStorage.removeItem('key')*
    Удаляет конкретную пару «ключ-значение» по указанному ключу.

- *localStorage.clear()*
    Полностью очищает всё хранилище для данного домена.


### 2. Типизация данных и JSON
**Важное техническое ограничение:** `localStorage` может хранить **только строки (Strings)**

Если попытаться сохранить объект или массив напрямую, JavaScript приведет его к строковому виду по умолчанию, и вы получите бесполезную запись `"[object Object]"`.

#### Решение: Сериализация и Парсинг

Для сохранения сложных структур данных (объектов, массивов) необходимо использовать формат **JSON**.

##### Сохранение (Сериализация):
Преобразуем объект в строку перед записью.

```JavaScript
const userSettings = { theme: 'dark', notifications: true };

// Преобразуем объект в JSON-строку
const jsonString = JSON.stringify(userSettings);

// Сохраняем строку
localStorage.setItem('settings', jsonString);
```


##### Чтение (Парсинг):
Получаем строку и преобразуем её обратно в объект.

```JavaScript
// Получаем JSON-строку
const rawData = localStorage.getItem('settings');

// Преобразуем обратно в объект JavaScript
const settingsObject = JSON.parse(rawData);
```


### 3. Проверка наличия данных
Метод `getItem` возвращает `null`, если запись с запрашиваемым ключом отсутствует. Это используется для проверки состояния (например, перед автозаполнением полей формы).

**Паттерн проверки:**

```JavaScript
const savedData = localStorage.getItem('register-data');

if (savedData !== null) {
    // Данные существуют: выполняем логику восстановления (парсинг, заполнение полей)
    console.log('Данные восстановлены из хранилища');
    const parsedData = JSON.parse(savedData);
} else {
    // Данных нет: инициализируем значения по умолчанию или оставляем поля пустыми
    console.log('Сохраненных данных нет');
}
```

> **Примечание:** В JavaScript `null` является _falsy-значением_, поэтому условие часто сокращают до `if (savedData) { ... }`.


### 4. Область применения
В реальных приложениях `localStorage` используется для временного хранения состояния интерфейса.

- **Подходит для:** Сохранения черновиков форм, цветовой темы, настроек отображения, токенов сессии (с оговорками по безопасности).

- **Не подходит для:** Долгосрочного хранения критически важных данных пользователя (они должны отправляться на сервер) и чувствительной информации (пароли), так как хранилище доступно из JavaScript.
