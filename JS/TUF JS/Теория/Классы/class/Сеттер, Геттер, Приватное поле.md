Пример
```JS
export default class Animal
{
    #age;
    #hunger;

    /**
     * @param {number} id     Уникальный идентификатор объекта
     * @param {string} name   Имя животного
     * @param {string} type   Тип животного (корова, курица, свинья)
     * @param {number} age    Возраст животного
     * @param {number} hunger Уровень голода [0-100]
     * @param {string} imgSrc Путь к изображению животного
     * @param {string} imgAlt Фраза, поясняющая что изображено на картинке
     */
    constructor(id, name, type, age, ageMax, hunger, imgSrc, imgAlt, hungerMin = 0, hungerMax = 100){

        this.id = id;
        this.type = type;
        this.name = name;
        this.age = age;
        this.ageMax = ageMax;
        this.hunger = hunger;
        this.hungerMin = hungerMin;
        this.hungerMax = hungerMax;
        this.isAlive = true;
        this.imgSrc = imgSrc;
        this.imgAlt = imgAlt;
    }

    set age(value)
    {
        const ageMin = 0;
        const {ageMax} = this;
  
        if(value > ageMin && value < ageMax)
        {
            this.#age = value;
        }

        else if(value >= ageMax) {
            this.#age = ageMax;
        }

        else if(value <= ageMin){
            this.#age = ageMin;
        }
    }

    get age()
    {
        return this.#age;
    }


    set hunger(value)
    {
        const {hungerMax, hungerMin} = this;
        if(value > hungerMin && value < hungerMax)
        {
            this.#hunger = value;
        }

        else if(value >= hungerMax)
        {
            this.#hunger = hungerMax;
        }

        else if(value <= hungerMin)
        {
            this.#hunger = hungerMin;
        }
    }

    get hunger()
    {
        return this.#hunger;
    }
}
```


Если бы у меня не было приватного поля, но был бы Сеттер для обычного
То есть не было бы поля `#hunger`, было бы просто `hunger`

То я бы обращался к нему внутри сеттера вот так `this.hunger`
Такое обращение вызывало бы сеттер снова, который в свою очередь тоже вызывал был сеттер, потому что при попытке изменения поля с Сеттером, вызывается этот самый Сеттер
Что бы этого избежать я сделал `hunger` приватным полем, что не вызывает сеттер

А так как `hunger` теперь приватный, то что бы обращаться к нему через `this.hunger` нужен ещё и Геттер


Из всего этого делаем вывод, сеттер и геттеры должны быть только у приватных полей


Так же, обратите внимание, в Конструкторе я обращаюсь не напрямую к приватному полю, а к сеттеру, что бы при создании объекта нельзя было присвоить не корректное значение
