### Как инициализировать переменные разными типами?
В JavaScript тип переменной определяется значением, которое вы в неё положили прямо сейчас. Это называется динамическая типизация.

Для создания переменных используются ключевые слова `let` (если значение будет меняться) и `const` (если не будет). `var` — устарело, не используйте его.

Вот как инициализировать основные типы:



``` JavaScript
// 1. Числа (Number) - в JS нет int или float, всё это Number
let count = 10;
let price = 99.99;

// 2. Строки (String) - можно одинарные или двойные кавычки
let firstName = "Ivan";
let lastName = 'Ivanov';

// 3. Булево (Boolean)
let isActive = true;
let isDone = false;

// 4. Массивы (Array) - в JS они резиновые (как List<T>)
let numbers = [];             // Пустой массив
let colors = ["Red", "Blue"]; // Массив с данными

// 5. Объекты (Object) - аналог классов/структур, но создаются на лету
let person = {
    name: "Alex",
    age: 25
};

// 6. Пустота (Null и Undefined)
let futureValue = null; // Мы явно сказали "тут ничего нет"
let notDefined;         // Мы объявили переменную, но ничего не дали (undefined)
```

#### Главное отличие от C#

В C# этот код вызвал бы ошибку компиляции, а в JS это норма:

``` JavaScript
let box = 100;      // Сейчас box — это число
box = "Сюрприз";    // Теперь box — это строка
box = [1, 2, 3];    // А теперь box — это массив
```

> **Совет:** Старайтесь не менять типы переменных по ходу работы, даже если язык это позволяет. Это приводит к трудноуловимым ошибкам. Если переменная создана для числа, пусть она всегда хранит число.

### Резюме для C#-разработчика

1. **Параметры функций:** Синтаксис их не проверяет. Используйте `Array.isArray()` для логики и **JSDoc** (`@param {Array}`) для удобства написания кода.
    
2. **Переменные:** Тип «приклеен» к значению, а не к имени переменной.
    
3. **Ключевые слова:** Забудьте `int`, `string`, `bool`. Используйте `const` (по умолчанию) и `let` (если нужно менять значение)
