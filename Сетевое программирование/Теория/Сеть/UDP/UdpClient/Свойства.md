#### Ttl
Это "счетчик прыжков"
Каждый раз, когда пакет проходит через маршрутизатор (роутер), этот счетчик уменьшается на 1
Когда он достигает 0, пакет уничтожается

- `Ttl = 1` *(часто по умолчанию)*: Пакет *не покинет* твой компьютер (или, в лучшем случае, твой сегмент локальной сети)
- Он не пройдет через роутер
- Это полезно, если ты хочешь, чтобы данные видели только программы на твоем ПК

- `Ttl > 1` *(например, 32)*: Пакет сможет пройти через несколько роутеров, охватывая большую часть твоей локальной сети.

Если приёмник и отправитель находятся в разных подсетях (например, подключены к разным Wi-Fi роутерам, которые соединены друг с другом), `Ttl = 1` не сработает
Нужно ставить значение побольше
`32` - это безопасное значение для большинства локальных сетей.
