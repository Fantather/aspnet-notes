#### Состояние Task после Cancel
Если выполнение метода было прервано из-за отловленного исключения `OperationCanceledException`, то состояние у Task будет - *Canceled*
Это специальный тип ошибки и  Планировщик потоков понимает что это была запланированная ошибка

Если исключение не отловить, то будет - *Faulted*

Если использовать `IsCancellationRequested` и `return`, то состояние у Task будет - *RunToCompletion*, ведь код просто завершил работу через `return`
Если обычное исключение было обработано, то статус Task тоже будет *RunToCompletion*


**Важно**
При передаче в Task метода который работает с *CancellationToken*, в конструктор `Task`, `Task.Run` или `Task.Factory` тоже нужно передавать этот же токен ([[Опции создания Task.Run#CancellationToken|Подробности]])
