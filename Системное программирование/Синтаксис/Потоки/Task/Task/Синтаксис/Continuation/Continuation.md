`.ContinueWith()` - запускает в том же потоке переданный метод после того как объект Task завершит свою работу
Традиционно используется для обработки результата этого метода

Обычно именно Continuation-методы используют для того что бы доставать исключения из других потоков, это обязательный процесс


Принимает Лямбду, Ссылку на метод или Делегат
Сигнатура метода должна принимать Task, передаёт в Continuation-метод объект Task, который только что завершил работу
Из него можно получить информацию о работе в другом потоке и результат этой работы
```C#
Task task = Task.Run(() => 
{
	/*...*/
});

task.ContinueWith(AnotherMethod);
task.ContinueWith(task => { /* ... */})
```

Более подробно о работе с возвращаемыми значениями смотри в [[Возврат значений и обработка исключений]] и [[Получение результата]]


#### TaskContinuationOptions
Это *enum*, который принимает `.ContinueWith()` и который позволяет нам задать условия, при котором `.ContinueWith()` должен сработать

- `TaskContinuationOptions.OnlyOnRanToCompletion` - сработает, только если задача завершилась успешно
- `TaskContinuationOptions.OnlyOnFaulted` - сработает, только если задача вернула необработанное исключение
- `TaskContinuationOptions.OnlyOnCanceled` - сработает, только если задача была отменена
