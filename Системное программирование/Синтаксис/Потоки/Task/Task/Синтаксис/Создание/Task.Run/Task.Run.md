Создаёт объект Task и сразу помещает его на выполнение в Пул потоков 

Этот способ создания *Task* рекомендуется Документацией, потому что он безопаснее чем `Task.Factory` или создание `Task` через `new` и его отложенный запуск через `task.Start`

`Task.Run` является Атомарной операцией и имеет множество оптимизаций
Он был специально создан, чтобы принудительно использовать `TaskScheduler.Default` ([[Планировщик задач .NET#TaskScheduler.Default|Пул Потоков]]) и игнорировать `TaskScheduler.Current`

Принимает делегаты, методы и лямбду

Создаёт и возвращает объект *Task*, через который можно управлять дальнейшим выполнением потока
```C#
Task task = Task.Run(() => 
{
	Console.WriteLine("Задача начала выполняться в отдельном потоке...");
	Thread.Sleep(2000); // Имитируем долгую работу
	Console.WriteLine("Задача завершена.");
});
```
