`await` - заставляет родительский поток ожидать завершения метода

Что бы метод стал асинхронным, нужно ключевое слово `async`
Оно говорит что результат этого метода должен быть автоматически завёрнут в Task и что внутри используется `await`
Если ничего не возвращается, то будет возвращён Task без параметров

Task возвращается всегда, потому что мы обязаны получать результат из фоновых потоков и выпускать оттуда возможные исключения

При это сами по себе ключевые слова `async` `await` не помещают функции в другие потоки

`await` запоминает окружающий UI context и если он есть, то после выполнения метода выполнение продолжится в UI-потоке

Если UI context нет, то выполнение продолжится в любом фоновом потоке
Вторая часть метода, после `await task;` будет уже в другом потоке


```C#
public async Task<int> SumAsync()
{
	return await Task.Run(() => Sum());
}

public void main(string[] args)
{
	int a = SumAsync().Result; // Блокировка Main
}
```
