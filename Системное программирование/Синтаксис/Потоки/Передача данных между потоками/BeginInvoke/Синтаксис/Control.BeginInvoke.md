В Windows Forms `Control.BeginInvoke` служит другой цели: *маршалинга* выполнения кода обратно в поток UI.

Любой элемент управления наследует метод `BeginInvoke`, который ставит делегат в очередь сообщений UI-потока
Это единственный безопасный способ обновить UI из фонового потока

**`Control.BeginInvoke`** - критически важен для безопасного обновления UI в WinForms из фоновых потоков, но в новых проектах предпочтительнее использовать `InvokeAsync`, который возвращает `Task` и полностью интегрируется с `async`/`await`



*Маршалинг* - это процесс переноса данных или вызова метода из одного контекста исполнения в другой
В контексте WinForms - это, в первую очередь, перенаправление выполнения кода из фонового потока обратно в UI-поток
