```C#
public class Example
{
	// Метод, который будет выполняться асинхронно
	public static int LongRunningCalculation(int a, int b)
	{
		return a * b;
	}
}

public static void Main(string[] args)
{
	// Создаем экземпляр делегата
	Func<int, int, int> calculator = Example.LongRunningCalculation;
	
	
	// Запускаем асинхронный вызов
	IAsyncResult asyncResult = calculator.BeginInvoke(6, 7, null, null);
	
	
	// Ожидаем завершения асинхронной операции
	asyncResult.AsyncWaitHandle.WaitOne();
	
	try
	{
		// Получаем результат
		int result = calculator.EndInvoke(asyncResult);
	}
	catch (Exception ex) {}
}
```

``` C#
IAsyncResult BeginInvoke(args1, args2, AsyncCallback callback, object state);
```
`args` - Аргументы, которые принимает делегат, их количество зависит от количества параметров, которые требует делегат

`callback` - Делегат `AsyncCallback`, указывающий на метод, который будет автоматически вызван после завершения работы асинхронной операции
*Callback-метод* имеет строгую сигнатуру
`void AsyncCallback(IAsyncResult ar)`

`Delegate.EndInvoke` метод который необходимо вызвать для получения результата или обработки возможных исключений

`state` - Это способ "прикрепить" к асинхронному вызову любую дополнительную информацию (контекст), чтобы потом можно было её получить в `AsyncCallback` (методе обратного вызова) и понять, какой именно вызов завершился или какие данные с ним связаны
Доступен через `IAsyncResult.AsyncState`


`BeginInvoke` - Возвращает `IAsyncResult`, который можно использовать для вызова `EndInvoke` и получения результата


```C#
object EndInvoke(IAsyncResult result);
```
`EndInvoke`
- Получает возвращаемое значение асинхронного метода и возвращает его
- Освобождает ресурсы, связанные с асинхронной операцией
- Может пробросить исключение, возникшее в асинхронном методе
