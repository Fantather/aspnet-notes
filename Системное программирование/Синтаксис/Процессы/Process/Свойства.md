##### BasePriority
Возвращает базовый приоритет процесса (целое число `int`)
Может принимать такие значения:
- 4 - Idle - низкий приоритет
- 8 - Normal - нормальный приоритет
- 13 - High - высокий приоритет
- 24 - RealTime - приоритет реального времени (максимальный)
  Используется системными процессами, мы не можем его использовать

##### CanRaiseEvents
Логическое значение, показывающее, может ли процесс генерировать события, связанные с его работой
Если значение `false`, то обработчики событий, назначенные объекту, вызываться не будут


##### EnableRaisingEvents
Получает или задаёт логическое значение:
Разрешено ли объекту Process генерировать события

##### ExitCode
Код завершения процесса, который он установил при выходе

##### ExitTime
Время завершения процесса

##### Handle
Дескриптор (указатель операционной системы) этого процесса

##### HandleCount
Количество открытых процессом дескрипторов

##### HasExited
Показывает, завершён ли процесс

##### Id
Уникальный идентификатор процесса (PID - process Identidier)

##### MachineName
Имя компьютера

##### MainModule
Главный модуль (исполняемый файл) процесса

##### MainWindowHandle
Дескриптор главного окна процесса (если у него есть окно)

##### MainWindiwTitle
Заголовок главного окна процесса (если у него есть окно)

##### Modules
Коллекция модулей (`ProcessModuleCollection`), загруженных процессом

##### PriorityBoostEnabled
Указывает, может ли процесс временно повысить свой приоритет, когда его главное окно получает фокус

##### PriorityClass
Получает или задаёт значение перечисления `ProcessPriorityClass`, определяющее класс приоритета процесса
- *Normal* - нормальный приоритет (по умолчанию для всех процессов)
- *Idle* - процесс выполняется только когда система простаивает
- *High* - выполняет критически важные операции, которым нужно немедленное выполнение
- *RealTime* - максимальный возможный приоритет в ОС
- *BelowNormal* - приоритет между Normal и Idle
- *AboveNormal* - приоритет между Normal и High

##### ProcessName
Имя процесса

##### Responding
Показывает, отвечает ли пользовательский интерфейс процесса на действия пользователя

##### StandardError
Поток для чтения сообщений об ошибках, выводимых приложением

##### StandardInput
Поток стандартного вывода для выполняемого приложения 

##### StandardOutput
Поток стандартного ввода для выполняемого приложения 

##### StartInfo
Объект `ProcessStartInfo` с набором параметров для запуска процесса
Передаётся при вызове метода `Start`
Важные свойства `ProcessSartInfo`
- *Arguments* - аргументы командной строки для запускаемого приложения.
- *CreateNoWindow* - логическое значение: создавать ли окно при запуске
- *EnvironmentVariables* - словарь (`StringDictionary`) переменных окружения
- *FileName* - путь к запускаемому приложению (или документу)
- *LoadUserProfile* - загружать ли профиль пользователя из реестра при запуске
- *Password* - пароль пользователя для запуска процесса
- *UserName* - имя пользователя, от чьего имени запускается процесс
- *WindowStyle* - стиль главного окна при запуске
- *WorkingDirectory* - рабочая папка процесса

##### Threads
Набор потоков, выполняемых в рамках текущего процесса.
