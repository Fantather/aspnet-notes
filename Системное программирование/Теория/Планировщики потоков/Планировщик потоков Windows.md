Это низкоуровневый компонент системы Windows, он управляет *Всеми потоками(Thread) Всех процессов в системе*
Именно он решает, какой поток будет выполняться на одном из ядер процессора в каждый конкретный момент времени

Он постоянно переключается между сотнями потоков, выделяя каждому "Квант" времени CPU
Он использует сложную систему приоритетов, что бы важные потоки (На пример системные) получали больше времени, чем фоновые

Мы можем сами указывать приоритет потоку(Thread) при его создании, но Планировщик Windows может не следовать этому приоритету
